# 修复 CRUD 生成器右侧预览区

## 问题根因分析

### Bug 1（核心）：前后端 SSE 协议不匹配
- **位置**：`CrudGeneratorStreamService.java` + `useCrudGenerator.js`
- **问题**：
  - 后端 `processChunk()` 将包含 `[STAGE:xxx]` 的 chunk 转为 `progress` 事件发出，**同时将 stage 标记从 chunk 内容中过滤掉**
  - 前端 `handleSSEChunk` 只把 `chunk` 事件的纯文本内容追加到 `rawContent`，因此 `rawContent` 里**永远没有** `[STAGE:xxx]` 标记
  - `parseAllContent` 用 `[STAGE:xxx]` 分段解析 `rawContent`，**永远解析不出任何内容** → 右侧预览为空

### Bug 2：Vue 响应式缺失
- **位置**：`useCrudGenerator.js` `displayContent`
- **问题**：`displayContent` 是 `ref({...})`，直接赋值 `displayContent[key] = val` 绕过了 Vue 的响应式，应改为 `displayContent.value[key] = val`

### Bug 3：流式内容分段写入逻辑缺失
- **位置**：`useCrudGenerator.js` `handleSSEChunk`
- **问题**：当前只在 `complete` 时做一次全量解析，没有在流式接收时按阶段实时写入右侧预览区

## Task 1：修复后端 SSE 流内容保留阶段标记

**文件**：`forge-plugin-generator/src/main/java/com/mdframe/forge/plugin/generator/service/CrudGeneratorStreamService.java`

**修改**：`processChunk()` 方法改为：当 chunk 同时包含 stage 标记和 JSON 内容时，**同时发送 progress 事件和 chunk 事件**；不再过滤 chunk 内容中的阶段标记（或额外发送一个带标记的 chunk）。

更好的方案：
- 当 chunk 包含 `[STAGE:xxx]` 时，先发 `progress` 事件，再把去掉标记后的剩余内容作为 `chunk` 事件发出（如果有内容的话）
- 在 `Flux.concat` 中改为使用 `flatMap` 以支持单个 chunk 触发多个 SSE 事件

```java
// 当前错误逻辑（只返回一个事件）：
private ServerSentEvent<String> processChunk(String chunk) {
    if (chunk.contains("[STAGE:searchSchema]")) {
        return buildProgressEvent(STAGE_SEARCH, "正在生成 searchSchema...");  // stage标记后的内容丢了！
    }
    ...
}

// 修复方案：改返回 Flux<ServerSentEvent<String>>，使用 flatMap
private Flux<ServerSentEvent<String>> processChunkToFlux(String chunk) {
    List<ServerSentEvent<String>> events = new ArrayList<>();
    // 匹配阶段标记并分别发送 progress + chunk
    if (chunk.contains("[STAGE:searchSchema]")) {
        events.add(buildProgressEvent(STAGE_SEARCH, "正在生成 searchSchema..."));
        chunk = chunk.replace("[STAGE:searchSchema]", "").trim();
    }
    // ... 其他阶段类似处理
    if (StringUtils.isNotBlank(chunk)) {
        events.add(buildChunkEvent(chunk));
    }
    return Flux.fromIterable(events);
}
```

同时修改 `streamGenerate()` 中的 `.map(this::processChunk)` 改为 `.flatMap(this::processChunkToFlux)`。

## Task 2：修复前端 SSE 分段解析逻辑

**文件**：`forge-admin-ui/src/composables/useCrudGenerator.js`

**修改 1**：`handleSSEChunk` 中增加阶段感知的实时写入：
```js
function handleSSEChunk({ event, data }) {
  if (event === 'progress') {
    currentStage.value = data.stage
    currentStageMessage.value = data.message
    updateLastAssistantMessage(data.message)
    // 切换阶段时切换右侧预览 tab
    if (data.stage.startsWith('generating-')) {
      const stageToFile = {
        'generating-search': 'searchSchema',
        'generating-columns': 'columnsSchema',
        'generating-edit': 'editSchema',
        'generating-api': 'apiConfig',
      }
      const fileKey = stageToFile[data.stage]
      if (fileKey) {
        activeFile.value = fileKey
        currentReceivingFile.value = fileKey  // 新增：记录当前接收的文件类型
      }
    }
  } else if (event === 'chunk') {
    rawContent.value += data.content
    // 实时追加到当前阶段的 displayContent
    if (currentReceivingFile.value) {
      displayContent.value[currentReceivingFile.value] = 
        (displayContent.value[currentReceivingFile.value] || '') + data.content
    }
  }
}
```

**修改 2**：修复 `displayContent` 响应式访问（所有 `displayContent[key]` 改为 `displayContent.value[key]`）：
- `startNewSession()` 中的 `displayContent.value = {...}` 已正确
- `parseAllContent()` 中赋值改为 `displayContent.value[stage] = ...`
- `generatedFiles.value[stage] = ...` 中同步更新

**修改 3**：`handleSSEComplete` 中调用 `parseAllContent` 做最终格式化（将流式拼接的原始内容做 JSON 美化），**同时保留实时预览**。

**修改 4**：`currentFileContent` 的 computed setter 修复：
```js
const currentFileContent = computed({
  get: () => displayContent.value[activeFile.value] || '',
  set: (val) => { displayContent.value[activeFile.value] = val }
})
```
（注意 `.value` 缺失）
