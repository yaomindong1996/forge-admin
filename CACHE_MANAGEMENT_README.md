# Redis缓存管理功能使用说明

## 功能概述

Redis缓存管理功能为系统提供了可视化的缓存监控和管理界面,支持查询、查看、删除缓存数据等操作。

## 功能特性

### 1. 缓存列表查询
- 支持通过缓存键名模糊搜索(支持通配符`*`)
- 分页展示缓存列表
- 显示缓存键、类型、值预览、过期时间等信息

### 2. 缓存详情查看
- 查看完整的缓存键名
- 根据缓存类型智能展示值内容:
  - **String类型**: 直接显示,自动识别JSON并格式化
  - **Hash类型**: 以表格形式展示键值对
  - **Set类型**: 以列表形式展示元素
  - **List类型**: 以索引表格形式展示元素
- 显示过期时间(TTL)

### 3. 缓存删除操作
- 删除单个缓存项
- 批量删除选中的缓存项
- 清空所有缓存(支持按模式匹配)

## 部署步骤

### 1. 后端部署

后端代码已自动集成到系统中,无需额外操作。相关文件:

- **接口扩展**: `ICacheService.java`
- **实现类**: `RedissonCacheServiceImpl.java`
- **DTO对象**: `CacheInfoDTO.java`
- **控制器**: `SysCacheController.java`

### 2. 前端部署

前端页面文件已创建:

- **页面文件**: `/forge-admin-ui/src/views/system/cache.vue`

### 3. 数据库配置

执行SQL脚本添加菜单权限:

```bash
# 1. 查询"系统管理"菜单的ID
SELECT id FROM sys_resource WHERE resource_name = '系统管理' AND resource_type = 1 LIMIT 1;

# 2. 编辑 /forge/forge-admin/sql/cache_menu.sql
#    将第25行的parent_id值替换为步骤1查询的结果

# 3. 执行SQL脚本
mysql -u用户名 -p密码 数据库名 < /path/to/cache_menu.sql

# 4. 为超级管理员分配权限(可选,如果超级管理员拥有所有权限则跳过)
# 执行以下SQL:
INSERT INTO sys_role_resource (tenant_id, role_id, resource_id, create_time)
SELECT 000000, 1, id, NOW() 
FROM sys_resource 
WHERE resource_name LIKE '缓存%';
```

### 4. 访问系统

1. 重启后端服务(如果后端有热部署则无需重启)
2. 刷新前端页面或重新登录
3. 在左侧菜单"系统管理"下找到"缓存管理"菜单
4. 点击进入即可使用

## 使用示例

### 查询所有缓存
1. 在搜索框输入`*`(默认值)
2. 点击"查询"按钮
3. 查看缓存列表

### 搜索特定前缀的缓存
1. 在搜索框输入`user:*`
2. 点击"查询"按钮
3. 只显示以"user:"开头的缓存

### 查看缓存详情
1. 在缓存列表中点击"详情"按钮
2. 在弹窗中查看完整的缓存信息
3. 根据类型查看格式化后的值内容

### 删除缓存
1. **单个删除**: 点击操作列的"删除"按钮
2. **批量删除**: 勾选多个缓存,点击工具栏"批量删除"按钮
3. **清空所有**: 点击工具栏"清空所有缓存"按钮(危险操作,需二次确认)

## API接口说明

### 分页查询缓存
```
GET /system/cache/page
参数:
  - pattern: 搜索模式(默认"*")
  - page: 页码(默认1)
  - pageSize: 每页大小(默认10)
```

### 获取缓存详情
```
POST /system/cache/getInfo
参数:
  - key: 缓存键名
```

### 删除缓存
```
POST /system/cache/remove
参数:
  - key: 缓存键名
```

### 批量删除缓存
```
POST /system/cache/removeBatch
Body: ["key1", "key2", "key3"]
```

### 清空缓存
```
POST /system/cache/clear
参数:
  - pattern: 匹配模式(默认"*")
```

## 安全说明

1. **权限控制**: 只有拥有`system:cache:list`权限的用户才能访问此功能
2. **操作日志**: 所有删除操作都会记录到操作日志中
3. **二次确认**: 删除和清空操作都需要用户二次确认
4. **危险提示**: 清空所有缓存操作会使用error级别的对话框提示

## 注意事项

1. **性能考虑**: 
   - 当缓存数量超过10000时,建议使用更精确的搜索模式
   - 避免频繁执行"清空所有缓存"操作

2. **数据安全**:
   - 删除缓存后无法恢复,请谨慎操作
   - 清空缓存可能影响系统运行,建议在测试环境或维护时间执行

3. **值显示限制**:
   - 列表中的值预览限制在100字符以内
   - 超大值在详情弹窗中可能加载较慢

## 故障排除

### 问题1: 菜单不显示
**解决方案**:
1. 检查SQL脚本是否执行成功
2. 检查角色是否有对应权限
3. 清空浏览器缓存并重新登录

### 问题2: 缓存列表为空
**解决方案**:
1. 检查Redis连接是否正常
2. 确认Redis中确实有数据
3. 检查搜索模式是否正确

### 问题3: 删除操作失败
**解决方案**:
1. 检查后端日志
2. 确认缓存键名正确
3. 检查Redis服务状态
