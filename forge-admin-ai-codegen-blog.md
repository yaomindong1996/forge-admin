# 我在 forge-admin 里搞了个 AI 代码生成器：0 代码生成业务逻辑，还能自定义模板

## 前言

做后端开发的同学应该都有这种体验：**80% 的时间都在写重复代码**。

增删改查、列表查询、分页、导出……每个业务模块几乎都是同一套流程，换个表名、换个字段名，复制粘贴改一改，一天能写十几个模块。但说实话，这种「体力活」真的很消磨开发热情。

之前 forge-admin 已经内置了可视化代码生成器，可以通过导入数据库表一键生成前后端 CRUD 代码。但最近我在想：**能不能更进一步？让 AI 来理解业务需求，根据自定义模板直接生成代码？**

于是，我在 forge-admin 里实现了一个 **AI 驱动的代码生成功能**，今天来分享一下这个功能的实现思路和使用体验。

## 这个功能能做什么？

先说结论：**你只需要描述业务需求，AI 就能根据你配置的业务模板生成完整代码，并且支持一键下载，直接复制到工程中使用。**

具体来说，它解决了这几个痛点：

- **告别手写 CRUD**：描述业务需求，AI 自动生成增删改查代码
- **模板可定制**：不同项目有不同的代码规范，你可以自定义业务模板，AI 按照你的模板风格生成代码
- **开箱即用**：生成的代码直接下载，复制到工程就能跑
- **0 代码门槛**：不需要写任何代码，只需要用自然语言描述需求

## 核心设计思路

整个功能的架构其实不复杂，核心就是三步：

```
用户描述需求 → AI 理解需求 + 读取业务模板 → 生成符合模板规范的代码
```

### 1. 业务模板配置

这是整个功能最关键的部分。每个团队都有自己的代码规范和项目结构，所以模板不能写死。

在 forge-admin 中，你可以配置自己的业务模板，包括：

- **代码结构模板**：Controller、Service、Mapper、Entity 等各层的代码模板
- **字段映射规则**：哪些字段需要索引、哪些字段需要校验
- **业务逻辑模板**：查询条件组合、关联关系、导出配置等

模板配置好之后，AI 生成的代码就会严格遵循你的项目规范，不需要二次调整。

### 2. AI 理解业务需求

用户只需要用自然语言描述业务需求，比如：

> "我需要一个商品管理模块，包含商品名称、价格、库存、分类、状态等字段，支持按分类和状态筛选，支持分页查询和导出"

AI 会解析这段描述，提取出：

- **实体信息**：字段名称、类型、校验规则
- **业务逻辑**：查询条件、排序规则、分页配置
- **接口设计**：需要哪些 API 接口

### 3. 代码生成与下载

AI 生成代码后，会按照你配置的模板结构组织代码文件：

```
├── controller/
│   └── ProductController.java
├── service/
│   ├── ProductService.java
│   └── impl/
│       └── ProductServiceImpl.java
├── mapper/
│   └── ProductMapper.java
├── entity/
│   └── Product.java
└── vo/
    ├── ProductVO.java
    └── ProductQueryVO.java
```

支持一键打包下载，下载后直接复制到你的工程对应目录即可。

## 和传统代码生成器的区别

你可能觉得，这不就是传统的代码生成器吗？其实差别还是挺大的：

| 对比项 | 传统代码生成器 | AI 代码生成 |
| --- | --- | --- |
| 输入方式 | 导入数据库表，手动配置字段 | 自然语言描述需求 |
| 模板能力 | 固定模板，修改成本高 | 自定义业务模板，灵活适配 |
| 业务逻辑 | 只能生成基础 CRUD | 能理解业务语义，生成复杂逻辑 |
| 学习成本 | 需要熟悉工具配置 | 会说话就行 |

传统代码生成器是「你告诉它怎么做」，AI 代码生成是「你告诉它你要什么」。

## 实际使用体验

我自己在项目中用了一段时间，说几个真实感受：

**效率提升明显**：以前写一个标准业务模块，从建表到写完前后端代码，至少要 1-2 个小时。现在描述需求 + AI 生成 + 下载复制，**10 分钟搞定**。

**代码质量稳定**：因为是按照模板生成的，代码风格统一，不会出现「这个模块是我写的，那个模块是别人写的，风格完全不一样」的问题。

**新人友好**：团队新来的同学不需要熟悉项目的代码规范，按照模板生成就行，降低了不少上手成本。

**模板复用**：模板配置好之后，团队里所有人都能用。换项目了，换个模板配置继续用。

## 技术实现要点

简单聊聊几个关键的实现点：

### 模板引擎设计

模板系统采用了**策略模式 + 模板引擎**的组合设计。每种代码类型（Controller、Service、Entity 等）对应一个模板策略，模板中支持变量占位符，AI 生成的内容会自动填充到对应位置。

### AI Prompt 工程

Prompt 的设计是这个功能的核心。需要让 AI 理解：

1. 用户的业务需求描述
2. 当前的模板结构和规范
3. 输出格式要求（JSON 结构化数据）

通过精心设计的 Prompt 模板，AI 能够稳定地输出符合预期的结构化数据，再由模板引擎渲染成最终代码。

### 代码下载

生成完成后，会将所有代码文件打包成 ZIP 格式，保持标准的 Maven/工程目录结构，下载后直接解压到对应目录即可使用。

## 适用场景

这个功能特别适合以下场景：

- **快速搭建新模块**：新项目启动，需要快速搭建多个业务模块
- **团队协作规范化**：统一代码风格，减少 code review 中的格式问题
- **降低开发门槛**：让初级开发者也能快速产出规范代码
- **重复性工作自动化**：把时间花在真正有价值的业务逻辑上

## 写在最后

AI 代码生成不是要取代开发者，而是把开发者从重复劳动中解放出来。**让 AI 做它擅长的事（模式匹配、代码生成），让人类做人类擅长的事（业务思考、架构设计）。**

forge-admin 的 AI 代码生成功能目前还在持续迭代中，后续计划支持：

- 更多代码模板类型（如前端页面模板）
- 智能代码审查建议
- 基于现有代码的反向模板学习
